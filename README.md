ğŸ›  Project Standards (Important!)
We use Conventional Commits. This means every commit message must follow a specific format, or it will be blocked by a git hook.

Commit Message Format:
type: description

Common types:

feat: A new feature (e.g., feat: add player movement)

fix: A bug fix (e.g., fix: resolve jumping glitch)

chore: Maintenance tasks (e.g., chore: update readme)

refactor: Code changes that neither fix a bug nor add a feature

If your commit is blocked, check your spelling and ensure you have a space after the colon!
